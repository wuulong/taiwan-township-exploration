# 第 2 章：鄉鎮的樣態與分類邏輯——以新竹市為 POC 試點

本章節作為《台灣鄉鎮探索導航》與 **W 系統** 深度對齊的「概念驗證 (POC)」。我們將透過新竹市的三個行政區，展現「書本方法論」如何精確映射至「數位地圖數據」。

## 2.1 方法論：從機能解構到分類標籤
在 W 系統中，每一個行政區不再只是一個幾何多邊形，而是具備「樣態 ID」與「屬性標籤」的智能實體。我們定義了以下通用協定：

*   **Category (分類)**：定義鄉鎮的機能靈魂。
*   **Tags (標籤)**：定義細節的經濟與人文維度。

## 2.2 新竹市：三區三樣態的實踐對齊

### 2.2.1 科技核心 (Tech Core)：新竹市東區
> **W 系統 ID**：[[TOWN_10018010]]
> **屬性注入**：`taxonomy.class = 'tech_core'`

**方法論解讀**：
東區是台灣半導體產業的震央。在 W 系統的深度地誌描述中提到：
> *「（東區）展現了科技新貴生活圈與百年老市場共生共榮的奇特景觀……高密度的科技人才社區取代了原有聚落，建構出全球化的產業網絡。」*

**書本與數據連結**：
當我們在書中討論「全球化對在地社區的重塑」時，我們可以直接引用「東門市場」與「竹蓮市場」的現代化與活化歷程作為證法，這些細節完整儲存於 W 系統的數位檔案中。

### 2.2.2 都會核心 (Urban Core)：新竹市北區
> **W 系統 ID**：[[TOWN_10018020]]
> **屬性注入**：`taxonomy.class = 'urban_core'`

**方法論解讀**：
北區匯集了新竹市最深厚的信仰與商務核心。其特徵在於以「城隍廟」為圓心的常民生活圈。

### 2.2.3 邊陲轉型 (Edge Town)：新竹市香山區
> **W 系統 ID**：[[TOWN_10018030]]
> **屬性注入**：`taxonomy.class = 'edge_town'`

**方法論解讀**：
香山區展現了從農業與玻璃工業轉向「生態旅遊」的轉型樣態。
> *「（香山）展現了從農業、玻璃漁業轉向濕地生態保育與文創遊程的轉變過程。」*

## 2.3 W 系統實踐：SQL 動態解析
本章節的方法論不只是文字，讀者可以直接在 W 系統中使用以下 **通用 SQL** 來驗證：

### 查詢特定樣態的行政區
```sql
SELECT feature_id, name, 
       json_extract(meta_data, '$.taxonomy.class') as category
FROM walking_map_features 
WHERE category = 'tech_core';
```

### 查詢具備特定文化特徵的標籤
```sql
SELECT name, description 
FROM walking_map_features 
WHERE json_extract(meta_data, '$.taxonomy.tags') LIKE '%culture:temple_clusters%';
```

## 2.4 通用軟體升級建議 (App Roadmap)
為了讓本書的 POC 效果最佳化，我們建議 WalkGIS App 實作以下通用邏輯：

1.  **語義標籤選單**：App 應能掃描 `taxonomy.class` 欄位，自動在地圖側邊欄生成下拉選單。
2.  **分類聯動顯示**：點擊「科技核心」時，地圖應自動隱藏非此類別的行政區，僅顯示特定色彩標出的目標區域。
