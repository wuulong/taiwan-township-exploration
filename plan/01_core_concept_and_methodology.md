# 台灣鄉鎮探索導航計畫：核心概念與方法論

## 1. 計畫起源與核心轉變
計畫的初衷源於對台灣流域的探索過程，進而體認到在地探索的本質即是「鄉鎮探索」。

*   **從流域到鄉鎮**： 流域探索雖具特色，但實際落地時，景點與餐飲食宿資訊皆隸屬於特定的鄉鎮行政區劃中，因此將核心調整為「台灣 368 鄉鎮」。
*   **架構前行**： 此計畫強調「先建構框架，再填充內容」。與其逐個點狀記錄，不如先建立涵蓋全台所有鄉鎮的完整架構。
*   **數據與方法論的深度結合**： 實體書不只是紙本紀錄，而是 `WalkGIS (WG)` 數位系統的「操作導引」與「解讀大腦」。

## 2. 三位一體的整合結構 (The Trinity)
計畫明確區分了書本、資料庫與 AI 技能的功能分工，形成一個具備生命力的探索生態系統。

### 2.1 實體書：方法論與解讀大腦
提供探索的「靈魂」：
*   **鄉鎮分類與樣態**： 分析不同鄉鎮的特性（如邊陲小鎮、大都市核心、科技衛星等）。
*   **動態內容引用**： 書本將資料庫中的 POI 視為自己的一部分。當書本提及特定區域時，讀者可透過 SQL 查詢或地圖連結獲取最新數據。
*   **機能發展與規劃**： 從國土規劃維度提供深層解讀。

### 2.2 數位系統 (WG/WIPS)：動態數據中心
利用現有的數位地圖與 SQL 資料庫 (DB)，處理具體的行政區劃與 POI。
*   **Meta Data 整合**： 鄉鎮的分類、Tag 與進度狀態將直接寫入 `walkgis.db` 的 `meta_data` 欄位。
*   **動態列表**： 書本提供對應的 SQL 命令範例，讓使用者能產出即時的統計報表或導航清單。

### 2.3 Antigravity Skill：AI 輔助探索助手
本計畫將隨書附贈專屬的 `antigravity skill`：
*   **Agentic AI 輔助**： 使用者下載 Skill 後，其 AI 代理程式將具備自動操作 `walkgis.db`、生成研究指令與自動撰寫探勘筆記的能力。
*   **資料預熱自律**： 在編寫書籍過程中，Agent 會遵循「資料先導」原則：若 POI 內容為空，將優先自動啟動 AI 厚化任務填入內容，或建立具備洞察力的預設佔位符，確保書本引用內容具備實質意義。

## 3. 鄉鎮分類與標籤系統 (通用連結協定)
為了實現書本方法論在地圖上的「可視化」，我們採用一套通用的 `taxonomy` 標籤協定。

| 屬性 | 說明 | 範例 |
| :--- | :--- | :--- |
| **Category** | 鄉鎮的核心樣態分類 | `ghost_town`, `tech_satellite`, `urban_core` |
| **Tags** | 具備特徵的細部標記 | `economy:agri`, `style:temple_clusters` |
| **Views (SQL)** | 儲存於 Map Meta 的動態視圖 | `SELECT * FROM features WHERE taxonomy.class = '...'` |

## 4. 數據處理進程與執行階段
計畫提出了一套標準化的資料處理流程，確保書本引用的每一筆數據都具備生命力：
1.  **有框架 (Framework)**： 建立行政與地理基礎架構。
2.  **資料預熱 (Data Priming)**： 在編寫對應章節前，若 POI 為空，先進行 AI 厚化填入「甜肉」。
3.  **有甜肉 (Content Filling)**： 確保 POI 描述具備被引用的價值。
4.  **有地圖 (Mapping)**： 完成地理資訊與 **W 系統** 的結合。
5.  **驗證 (Verify)**： 實地校核與內容確認。

## 5. 技術實現與平台整合 (W 系統通用邏輯)
在本計畫中，**「W 系統」** 是一個核心術語，其完整涵蓋範圍包括：
*   **鄉鎮導覽圖層 (The Map)**：在 WalkGIS 平台上運行的「鄉鎮導航」管理圖層。
*   **鄉鎮與 POI 節點**：包含全台 368 鄉鎮及其關聯的所有食宿景點數據。
*   **後端資料庫 (walkgis.db)**：儲存幾何、Meta Data 與描述說明的核心引擎。

### 5.1 深度連結機制
*   **Map-Level Insights**： 在 `walking_maps` 中定義可執行的 `views` 標籤，讓 App 自動生成分析選單。
*   **Feature-Level Metadata**： 在 POI 中同步分類標籤與 W 系統數據。
*   **引證式撰寫**： 書本直接讀取 `features/` 中的片段，讓方法論在 W 系統中「被實踐」。

## 6. 核心觀點與金句
*   **關於架構**：「現在的精神是架構前行嘛，那你既然是一個一個鄉鎮的探索，那你的架構就是整個全部的鄉鎮。」
*   **關於定位**：「書本的目錄主要就是在建構這個方法論……肉（實質內容）就隨著這個 WG 的 map 持續的更新。」
*   **關於順序**：「先寫會經過的鄉鎮，就有點像是倒著寫，一開始架構有，然後就倒著寫。」
